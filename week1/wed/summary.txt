오디오/비디오 파일 삽입하기
    오디오 파일 종류 : mp3, ogg, wav 등
    <audio> 태그를 활용
        속성
        1. src : 오디오 파일의 위치
        2. controls : 재생 제어기 표시
        3. loop : 반복 재생
        4. autoplay : 문서가 출력됨과 동시에 자동 재생(대부분의 브라우저에서 막혀있음)
        5. muted : 음소거 상태로 시작
        6. preload : 재생 전에 미리 다운로드
    
    보조 태그 : <source>
        예시>
        <audio controls>
            <source src="bear.mp3" type="audio/mpeg">
            <source src="bear.ogg" type="audio/ogg">
            <source src="bear.wav" type="audio/wav">
            오디오파일을 재생할 수 없습니다
        </audio>
        크롬은 셋다 지원함

        1. bear.mp3를 재생
        2. 만일 브라우저에서 mp3를 재생할 수 없으면, bear.ogg를 재생
        3. 만일 브라우서에서 ogg를 재생할 수 없으면, bear.wav를 재생
        4. 만일 모두 재샐할 수 없으면, 마지막 문장을 출력

영역 처리(시멘틱)
    공간 분할 태그(한 페이지의 영역 구분)
    1. div - 브라우저 전체 공간을 기준으로 일정 영역을 설정하는 태그 요소, 블록 요소
            크기 지정 가능
    2. span - 브라우저의 일부 공간(블록 요소의 일부)을 설정하는 태그 요소, 인라인 요소
                크기 지정 불가능(css로 조정)
    3. iframe - 한 페이지를 여러 html로 구성
        (iframe은 html을 여러개 묶어서 사용할 수 있음)

시맨틱 태그(Semantic tag)
    페이지 영역에 의미를 부여하는 태그
    CSS가 적용되지 않을 경우 특별한 모양이나 형식을 보이지 않음
    페이지 완성 또는 작업 중, 유지보수 등의 작성 시
    해당 영역을 쉽고 빠르게 찾아서 처리하는데 도움을 줌
    
    태그 종류
        1)header : 머리글
        2)nav : 메뉴
        3)section : 본문 영역
        4)article : 세부 영역
        5)aside : 부가 정보, 광고
        6)footer : 바닥글

<hr> 태그
    수평선(분류선)
    사용 속성
        1) align : 정렬. left, center, right
        2) noshade : 입체감 제거
        3) size : 두께 지정, px(픽셀단위)
        4) width : 가로길이(너비). px, %

입력 양식(from, input 등)
    <form> 태그 : 사용자의 입력을 다른 페이지나 서버 프로그램에 전달하기 위한 태그
    주요 속성
        1) action : 입력 값을 받을 대상을 지정
                (대상 : jsp, asp, php, 프로그램)
        2) method : 전달 방식을 지정. get/post
                    기본값은 get

    전달 방식의 차이
        1) GET : 주소창에 전송  데이터가 노출되는 방식
                 보안상으로 문제가 없는 짧은 데이터 전송에 적합
        2) POST : 주소창에 데이터가 노출되지 않는 방식
                 길고 보안상으로 문제가 잇는 데이터 전송에 적합
